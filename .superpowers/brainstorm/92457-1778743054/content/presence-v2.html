Presence 域（修订）：Push 内模块，proto 独立

Presence 不单独部署——和 Push 同进程，共享心跳。proto 文件独立，方便未来拆分。

运行时架构
┌──── Push 进程 ──────────────────────────────┐
│                                               │
│  WS 长连接层                                   │
│  ├─ 连接管理                                  │
│  ├─ 心跳处理 ─────────┐                       │
│  ├─ 消息推送          │                       │
│  └─ ACK/重传          │  共享心跳，无 RPC     │
│                        ▼                       │
│  Presence 模块（同进程）                       │
│  ├─ 在线状态管理 (ONLINE/AWAY/BUSY/...)   │
│  ├─ 输入中指示                                │
│  ├─ 状态变化推送 (Subscribe 模式)             │
│  └─ 最后在线时间                              │
│                                               │
│  Redis: im:online:{uid} (路由，Push 管)        │
│  Redis: im:presence:{uid} (状态，Presence 管)  │
│                                               │
└───────────────────────────────────────────────┘

客户端                     Push 服务
  │                          │
  │── WS CLIENT_HEARTBEAT ──>│
  │                          │── 本地更新 im:online:{uid} + im:presence:{uid}
  │                          │   （一次心跳完成，无 RPC 开销）
  │                          │
  │── TypingIndicator WS ───>│── 本地 Presence.setTyping()
  │                          │── 推送给会话其他在线成员
  │                          │
  │<── PresenceChange NOTIFY ─│── Subscribe 事件触发

调整后的 Presence Service（proto 独立，不独立部署）

// presence/presence_service.proto — proto 文件独立，运行时在 Push 进程内
service PresenceService {
  // === 客户端直接调用的 RPC ===
  rpc SetPresence(SetPresenceReq) returns (SetPresenceRsp);
  rpc GetPresence(GetPresenceReq) returns (GetPresenceRsp);
  rpc BatchGetPresence(BatchGetPresenceReq) returns (BatchGetPresenceRsp);

  // === 订阅（客户端通过 WS 订阅好友状态变化） ===
  rpc SubscribePresence(SubscribeReq) returns (SubscribeRsp);
  rpc UnsubscribePresence(UnsubscribeReq) returns (UnsubscribeRsp);

  // === 输入中指示 ===
  rpc SendTyping(TypingReq) returns (TypingRsp);

  // 注意：无 Heartbeat RPC——心跳在 Push 进程内本地处理
  //       Push 收到 WS 心跳 → 直接更新 Presence 内存 + Redis
}
