Key Design Decisions

What improves

  • base.proto split → common/types.proto stays under 80 lines
  • Internal messages isolated in message_internal.proto
  • Each domain imports only what it needs from common/
  • Presence domain extracted from push — now a first-class service
  • Media consolidates file + speech (both are "attachment" concerns)
  • Identity separates auth logic from profile logic within one file

Migration cost

  • proto reorganization → all import paths change
  • Generated code paths shift → CMakeLists.txt updates
  • user → identity rename touches gateway routing
  • file + speech → media merges two small services
  • Recommend: new imports, deprecate old, then remove
